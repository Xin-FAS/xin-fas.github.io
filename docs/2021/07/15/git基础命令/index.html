git提交结构

  1. 工作区（就是本地初始化地方的文件）
  2. 暂存区（add 中的文件）
  3. 本地库（commit 打上标签的文件）
  4. 远程库（github或gitee仓库）

提交流程

  1. 首次利用git提交需要配置好全局用户信息
  1. 初始化仓库
  1. 本地文件提交到暂存区
  1. 暂存区加入本地库
  1. 建立本地链接
  1. 提交对应分支

问题：

add 后提示warning

原因：在 Unix 系统中，行尾用换行符 (LF) 表示。在windows中，一行用回车符（CR）和换行符（LF）表示（CRLF）。当您从 git 获取从 unix 系统上传的代码时，它们将只有一个 LF。

但这个问题不影响操作，再次add就正常了
