了解

用过element-ui的人肯定对这个指令的作用不陌生，既然要手动实现，我们就要先了解下，在element-ui中，他们是如何做好的

分析准备

把element-ui利用npm下载后，在依赖中找到源码

源码挺少，所以先别怕（对我自己说的），一步步分析下

然后就有大聪明（我）就会懂了，哈！，我直接在页面中引入，然后打个断点，再利用debugger不就可以调试源码了！

嗯！很可惜这是不行的，因为源码虽然在这，但是element使用的是编译后的代码，也就是lib文件夹下的代码，所以无效，所以正确调试流程应该如下：

  1. 下载element-ui源码，git clone https://github.com/ElemeFE/element.git（建议直接下zip，拉取太慢）
  2. npm i安装完成后在对应地点使用debugger或console.log
  3. npm run dist打包后把lib文件夹替换到你的项目中，运行查看

让我们开始！

已准备：

一个页面，引入clickoutside并使用

一个element-ui的源文件https://github.com/ElemeFE/element.git ，并且已经npm i安装完依赖

高版本node会报错，建议使用12.22.12，安装完依赖后运行npm run dist生成lib文件夹直接替换到项目中（项目需要先停止运行），然后再次运行时，以上页面功能正常说明源码编译正常，可以继续

on 方法

首先来看这on的作用

在源文件中找到element-ui/src/utils/dom.js文件，搜索export const on =就可以找到，如下：

可以看到这是一个立即执行函数，里面存在变量isServer（是否为服务端，可以视为false，具体见下面补充）

补充isServer：Vue.js 是一个用于构建客户端应用的框架，这个属性通常用于在服务端渲染（SSR）时区分客户端和服务端环境。我们并不使用SSR，所以可以视为false

所以上面代码就简单了，如果为服务端并且document.addEventListener存在，那就直接返回一个方法，方法接收三个参数（元素，事件名，处理函数）作用是为元素建立监听事件，并且addEventListener的第三个参数为false（默认值也是false，表示冒泡事件）

第三个参数详情：https://developer.mozilla.org/zh-CN/docs/Web/API/EventTarget/addEventListener

不懂还可以见：https://blog.csdn.net/SunDaDa9/article/details/103693062

那么不存在document.addEventListener时呢，也是返回一个作用和参数一样的方法，只不过为了兼容IE把addEventListener换成了attachEvent

补充：addEventListener为W3C标准，除IE浏览器使用，而attachEvent只能在IE浏览器上使用，语法：Element.attachEvent(Etype,EventName)，事件名前加on，如onclick
